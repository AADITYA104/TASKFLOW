4.1 Backend (Node.js + Express) 
• server/index.js — Express app setup, CORS, MongoDB connect 
• routes/auth.js — POST /api/auth/register, POST /api/auth/login 
• routes/projects.js — GET/POST/PUT/DELETE /api/projects 
• routes/tasks.js — GET/POST/PUT/DELETE /api/tasks 
• middleware/auth.js — JWT verify middleware 
• middleware/role.js — Admin only route protection 
 
4.2 Database Models (MongoDB + Mongoose) 
• User model — name, email, password (bcrypt hashed), role (admin/member) 
• Project model — name, description, color, dueDate, createdBy, members 
• Task model — title, desc, status, priority, assignedTo, project, dueDate 
 
4.3 Frontend API Integration 
• app.js — Replace all localStorage calls with real fetch() API calls 
• Handle JWT token in headers for authenticated requests 
• Error handling for failed API calls 
 
4.4 Deployment on Render.com (Free) 
• Create Render.com account (free tier available) 
• Deploy backend as Web Service 
• Set environment variables (MONGO_URI, JWT_SECRET) 
• Update frontend API_URL to point to live backend 
• Get live URL for submission
